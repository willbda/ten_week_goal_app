📋 Goals Resource

Manage goals with SMART criteria (Specific, Measurable, Achievable, Relevant, Time-bound)

GET /goals

Retrieve all goals with optional filtering

Query Parameters

has_dates (boolean) - Filter for time-bound goals
has_target (boolean) - Filter for measurable goals

Example Response 200

{
  "goals": [
    {
      "id": 1,
      "description": "Run 120km",
      "measurement_unit": "km",
      "measurement_target": 120.0,
      "start_date": "2025-10-10T00:00:00",
      "end_date": "2025-12-19T00:00:00",
      "type": "Goal"
    }
  ],
  "count": 1
}
curl "http://localhost:5001/api/goals?has_dates=true"
GET /goals/{id}

Retrieve a single goal by ID

Responses

200 Goal object

404 Goal not found

curl http://localhost:5001/api/goals/1
POST /goals

Create a new goal

Request Body

description (string, required) - Goal description
measurement_unit (string) - Unit of measurement
measurement_target (float) - Target value
start_date (datetime) - ISO format
end_date (datetime) - ISO format 
curl -X POST http://localhost:5001/api/goals \
  -H 'Content-Type: application/json' \
  -d '{
    "description": "Run 120km",
    "measurement_unit": "km",
    "measurement_target": 120.0,
    "start_date": "2025-10-10T00:00:00",
    "end_date": "2025-12-19T00:00:00"
  }'
PUT /goals/{id}

Update an existing goal (partial updates supported)

curl -X PUT http://localhost:5001/api/goals/1 \
  -H 'Content-Type: application/json' \
  -d '{"measurement_target": 150.0}'
DELETE /goals/{id}

Delete a goal (with automatic archiving)

curl -X DELETE http://localhost:5001/api/goals/1
GET /goals/{id}/progress

Get goal with detailed progress metrics and matching actions

Example Response 200

{
  "goal": { ... },
  "progress": {
    "total_progress": 102.5,
    "target": 120.0,
    "percent": 85.4,
    "remaining": 17.5,
    "is_complete": false,
    "is_overachieved": false,
    "matching_actions_count": 23,
    "unit": "km"
  },
  "matches": [
    {
      "action_id": 5,
      "action_description": "Morning run",
      "contribution": 5.2,
      "assignment_method": "unit_match",
      "confidence": 0.95
    }
  ]
}
curl http://localhost:5001/api/goals/1/progress

⚡ Actions Resource

Log and track actions with measurements, duration, and automatic goal matching

GET /actions

Retrieve all actions with optional filtering

Query Parameters

has_measurements (boolean) - Filter for actions with measurements
has_duration (boolean) - Filter for actions with duration tracking
start_date (date) - Filter actions after this date (ISO format)
end_date (date) - Filter actions before this date (ISO format) 
curl "http://localhost:5001/api/actions?has_measurements=true&start_date=2025-10-01"
GET /actions/{id}

Retrieve a single action by ID

curl http://localhost:5001/api/actions/1
POST /actions

Create a new action

Request Body

description (string, required) - Action description
measurements (object) - Measurements dict (e.g., {"distance_km": 5.0})
duration_minutes (float) - Duration in minutes
start_time (datetime) - ISO format
log_time (datetime) - Defaults to now 
curl -X POST http://localhost:5001/api/actions \
  -H 'Content-Type: application/json' \
  -d '{
    "description": "Morning run in the park",
    "measurements": {"distance_km": 5.2},
    "duration_minutes": 32.5,
    "start_time": "2025-10-14T07:00:00"
  }'
PUT /actions/{id}

Update an existing action

curl -X PUT http://localhost:5001/api/actions/1 \
  -H 'Content-Type: application/json' \
  -d '{"measurements": {"distance_km": 6.0}}'
DELETE /actions/{id}

Delete an action (with automatic archiving)

curl -X DELETE http://localhost:5001/api/actions/1
GET /actions/{id}/goals

Get goals matched to this action using inference logic

Example Response 200

{
  "action": { ... },
  "matched_goals": [
    {
      "goal_id": 1,
      "goal_description": "Run 120km",
      "contribution": 5.2,
      "assignment_method": "unit_match",
      "confidence": 0.95
    }
  ],
  "match_count": 1
}
curl http://localhost:5001/api/actions/5/goals

💎 Values Resource

Define and manage personal values with priority levels and life domain organization

GET /values

Retrieve all values with optional filtering

Query Parameters

type (string) - Filter by type: 'major', 'highest_order', 'life_area', 'general'
domain (string) - Filter by life domain 
curl "http://localhost:5001/api/values?type=major&domain=Health"
GET /values/{id}

Retrieve a single value by ID

curl http://localhost:5001/api/values/1
POST /values

Create a new value

curl -X POST http://localhost:5001/api/values \
  -H 'Content-Type: application/json' \
  -d '{
    "value_name": "Health and vitality",
    "value_type": "major",
    "life_domain": "Health",
    "priority": 85
  }'
PUT /values/{id}

Update an existing value

curl -X PUT http://localhost:5001/api/values/1 \
  -H 'Content-Type: application/json' \
  -d '{"priority": 90}'
DELETE /values/{id}

Delete a value (with automatic archiving)

curl -X DELETE http://localhost:5001/api/values/1

📅 Terms Resource

Manage time-bounded planning periods (10-week terms) with goal organization and progress tracking

GET /terms

Retrieve all terms with status and metrics

Query Parameters

status (string) - Filter by status: 'active', 'upcoming', 'complete' 
curl "http://localhost:5001/api/terms?status=active"
GET /terms/{id}

Retrieve a single term with metrics

curl http://localhost:5001/api/terms/1
GET /terms/active

Get the currently active term with metrics

Example Response 200

{
  "term": {
    "id": 3,
    "term_number": 3,
    "start_date": "2025-10-10T00:00:00",
    "end_date": "2025-12-19T00:00:00",
    "theme": "Health & Learning",
    "term_goal_ids": [1, 2, 3]
  },
  "status": "active",
  "days_elapsed": 4,
  "days_remaining": 66,
  "progress_percent": 5.7,
  "committed_goal_count": 3,
  "overlapping_goal_count": 2
}
curl http://localhost:5001/api/terms/active
POST /terms

Create a new term

Request Body

term_number (int, required) - Sequential term identifier
start_date (datetime, required) - ISO format
end_date (datetime) - Defaults to start_date + 70 days
theme (string) - Focus area for this term
term_goal_ids (list) - Goal IDs assigned to this term
reflection (string) - Post-term reflection notes 
curl -X POST http://localhost:5001/api/terms \
  -H 'Content-Type: application/json' \
  -d '{
    "term_number": 4,
    "start_date": "2025-12-23T00:00:00",
    "theme": "New Year Focus",
    "term_goal_ids": [1, 3, 5]
  }'
PUT /terms/{id}

Update an existing term

curl -X PUT http://localhost:5001/api/terms/1 \
  -H 'Content-Type: application/json' \
  -d '{"theme": "Updated Theme", "reflection": "Great progress made"}'
DELETE /terms/{id}

Delete a term (with automatic archiving)

curl -X DELETE http://localhost:5001/api/terms/1
POST /terms/{id}/goals

Add a goal to this term's commitment list

Request Body

goal_id (int, required) - Goal ID to add to term 
curl -X POST http://localhost:5001/api/terms/3/goals \
  -H 'Content-Type: application/json' \
  -d '{"goal_id": 7}'
DELETE /terms/{id}/goals/{goal_id}

Remove a goal from this term

curl -X DELETE http://localhost:5001/api/terms/3/goals/7
GET /terms/{id}/progress

Get detailed term progress with committed goals, actions, and metrics

Example Response 200

{
  "term": {...},
  "status": "active",
  "time_progress": {
    "days_elapsed": 4,
    "days_remaining": 66,
    "percent_complete": 5.7
  },
  "committed_goals": [...],
  "overlapping_goals": [...],
  "actions_in_term": [...],
  "counts": {
    "committed_goals": 3,
    "overlapping_goals": 2,
    "actions": 15
  }
}
curl http://localhost:5001/api/terms/3/progress

⚠️ Error Handling

HTTP Status Codes

200 Success - Request completed successfully
201 Created - Resource created successfully
400 Bad Request - Validation error or invalid parameters
404 Not Found - Resource doesn't exist
500 Server Error - Internal server error

Error Response Format

{
  "error": "Description of what went wrong"
}

Example Error

{
  "error": "Field 'description' is required"
}
